# Douleurs sous-servies (à relire avant usage)

> ⚠️ Fichier d'angles de message. Soit rédigé à la main, soit (re)généré par
> `discover_pains.py` (social listening). **À RELIRE et valider avant qu'il ne
> serve à la rédaction** (roadmap #5). Ces angles ne servent JAMAIS à collecter
> des contacts : les destinataires restent les opt-in de la table `email`
> (Règle d'Or #4).

Format d'une entrée :

### Pricing d'exotiques sans code de référence
- **Douleur** : recoder un moteur (Monte-Carlo, arbres, PDE) à la main à chaque
  produit, sans base de code documentée à laquelle se référer.
- **Signal** : récurrent dans les communautés quant ; beaucoup réinventent la
  roue ou copient du code non vérifié.
- **Produit FQ** : FQ-KERNEL (accès au code source, documenté).
- **Angle/accroche** : « pricer une exotique sans réécrire ton moteur à chaque
  fois ».

### Vol implicite / surfaces de vol : appliquer sans comprendre
- **Douleur** : utiliser des formules de vol implicite comme une boîte noire,
  sans maîtriser la construction de la surface.
- **Signal** : questions récurrentes, confusions fréquentes.
- **Produit FQ** : KERNEL (code lisible) + matière pédagogique FrenchQuant.
- **Angle/accroche** : « la surface de vol, lue ligne par ligne ».

### Infra de backtest fiable et reproductible
- **Douleur** : backtests bricolés, non reproductibles, data sale.
- **Produit FQ** : KERNEL + Cloud Infra / Terminal selon le besoin.
- **Angle/accroche** : « un backtest dont tu peux relire chaque ligne ».
